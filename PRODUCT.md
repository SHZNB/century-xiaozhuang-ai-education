# Product

## Register

product

## Users

南京晓庄学院教师、校友、平台管理员和校庆项目预审人员。教师在备课、写教案、做课件、读长文档和整理教学资源时使用；管理员在上线前维护 100 个教育智能体、模型路由、校友认证和 Token 账本；预审人员需要快速判断平台是否完整、可信、可部署。

## Product Purpose

百年晓庄智慧教育平台是面向南京晓庄学院百年校庆与智慧教育建设的智能体集成平台。它把教育智能体目录、AI 教育助手、教学工作流、校友认证权益和后台治理集中到一个可演示、可部署、可迁移到 GitHub Pages 或 Node 服务端的项目骨架中。成功标准是：打开即能预审核心功能，教学任务入口清晰，100 个智能体目录完整，受保护模块按需认证，正式上线前可替换为学校授权素材和统一身份认证。

## Brand Personality

庄重、可信、务实。界面应有百年校庆的红金历史感，但工作区域必须克制、高对比、可扫描，避免把教学工具做成宣传海报。

## Anti-references

不做通用 SaaS 紫蓝渐变，不做玻璃拟态堆叠，不做大量装饰光球，不使用假校徽或自创校标，不让 AI 生成图冒充真实校园影像。功能页避免宣传型大卡片和空洞口号，优先呈现任务、状态、路由和可操作入口。

## Design Principles

1. 百年气质服务任务，不压过任务。
2. 每个入口都说明用户能完成什么，而不是展示系统有多复杂。
3. 模型路由、校友权益、智能体状态要可见、可追踪、可解释。
4. 管理与预审界面优先密度、对齐和筛选效率。
5. Demo 可以模拟数据，但认证、额度、发布状态和受保护入口的交互必须可信。

## Accessibility & Inclusion

默认按 WCAG AA 对比度设计；移动端 375px 宽度不可横向溢出；所有可点击项有明确 hover/focus 状态；动效控制在 150-300ms，并尊重 `prefers-reduced-motion`；颜色不得作为唯一状态表达。
