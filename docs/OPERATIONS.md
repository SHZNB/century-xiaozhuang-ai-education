# 平台运营工作流

## 智能体上线

1. 负责人在“应用管理”中新建草稿。
2. 完成名称、领域、简介、Logo、提示词和服务端连接别名。
3. 配置系统提示词和首选模型路由；平台对话型智能体会进入统一模型网关。
4. 使用后台单个或批量测试入口验证草稿智能体的输入、正常输出、异常提示和模型路由；测试记录写入审计日志，不扣校友额度。
5. 内容审核人员确认版权、隐私、事实准确性与适用对象。
6. 智能体名称、Logo、接入方式、外部地址、API 地址、服务端别名、模型路由或系统提示词发生变化后，服务端会自动清除之前的通过测试状态；外部网页、嵌入页和 API 型智能体必须重新测试通过后才能从草稿或下架状态发布。
7. 管理员发布，师生端才可检索和使用；每次创建、更新、发布、下架和版本恢复都会保存版本快照。
8. 每月复查使用量、失败率和用户反馈；不合格应用下架整改。

## 智能体导入导出

1. 导出文件只包含目录配置，不得包含任何 API Key。
2. 服务端导入支持替换或合并策略，导入时会剥离 `apiToken`、`secret`、`key` 等敏感字段。
3. 导入后每个智能体都会生成版本快照，方便追溯批量变更来源。

## 校友认证

1. 校友提交基础信息和证明。
2. 若已配置校友会联合认证接口，服务端先调用接口核验；核验通过即自动发放权益。
3. 未配置接口、接口异常或核验未通过时进入人工审核。
4. 审核员仅通过私有临时链接查看证明。
5. 通过前核对姓名、毕业年份、院系专业与重复领取记录。
6. 审核通过后，服务端事务发放 100 万国产模型 Token。
7. 权益有效期为批准后连续 30 天。
8. OpenAI 与 Gemini 不从赠送额度中扣减。

## 模型管理

1. 超级管理员在服务端密钥中心配置厂商密钥。
2. 为各能力配置首选与备用模型。
3. 每次调用记录模型、路由原因、延迟、Token 与结果状态。
4. 首选模型故障时自动切备用；失败请求不扣用户额度。
5. 密钥轮换后无需修改或重新发布前端。
6. `MODEL_MAX_OUTPUT_TOKENS` 控制单次最大输出；国产模型调用会先原子预留额度，成功后按实际用量结算，失败或超时则释放。
7. 管理员可在后台逐个测试模型提供方连接，测试不走备用模型、不扣用户额度，并写入审计日志。
8. “模型调用台账”记录每次调用的状态、实际路由、备用链路、耗时、Token 和关联智能体/工作流；它不保存用户提示词、模型回答或密钥，可用于排查供应商故障和复盘额度结算。

## 教学工作流维护

1. 工作流模板首次由服务端 `server/workflows.mjs` 初始化，之后在管理后台维护名称、首选模型、成果类型、系统提示词、输入字段、输出结构、质量检查清单和发布状态。
2. 首页任务卡片只触发工作流 ID，不在浏览器保存真实工作流逻辑或密钥。
3. 管理员可在后台新建、导入、导出、编辑提示词、编辑输入/输出/质检规范、发布、停用、删除、查看版本记录和恢复历史版本。
4. 新增或调整工作流后，管理员应使用后台单个或批量测试入口确认提示词、模型路由和模型响应；测试会逐条写入 `workflow.test` 审计、模型调用台账和工作流最近测试状态，并进入上线自检，但不扣校友额度、不写普通会话、不生成成果文件。
5. 新增或调整工作流后需要补充一条服务端测试，确认路由、成果类型、停用状态和失败不扣费。
6. 工作流标题、摘要、首选模型、成果类型、系统提示词、输入字段、输出结构或质量检查清单发生变化后，服务端会自动清除之前的通过测试状态；工作流必须重新测试通过后才能从草稿或下架状态发布。
7. 工作流执行写入会话、审计日志和 Token 账本，便于后续统计飞象老师式教学任务使用情况。
8. 工作流成果会保存为服务端资产并提供受权限保护的下载链接；网页动画成果下载为 ZIP 包，普通教学成果下载为 HTML 文件。生产环境迁移到 OSS 时保持 `/api/artifacts/:id/download` 契约不变。
9. 用户可在“成果库”中按关键词、成果类型和工作流筛选，并查看、下载、删除或批量删除自己通过工作流生成的教案、课件、题目、长文档摘要和网页动画；应用管理员可查看并批量删除全平台最近成果用于运维核验和空间清理，所有删除都会写入 `artifact.delete` 审计日志。
   筛选后，成果库资产治理概览会同步显示当前范围内 HTML/ZIP 数量、主要工作流分布、已选数量和最新成果时间，便于迁移 OSS、清理旧文件或核对空间占用。
10. 超级管理员可在“成果库”中导出成果资产清单 CSV，用于迁移到 OSS、核对空间占用和按工作流盘点成果；清单只包含元数据，不包含模型生成正文，导出写入 `artifact.export` 审计日志。
11. 超级管理员可在“成果库”中按保留天数批量清理旧成果，服务端会同步删除成果文件和元数据，并写入 `artifact.cleanup` 审计日志；生产迁移到 OSS 后应保持同样的权限和审计语义。

## 运营统计

1. 管理后台可切换近 7 天、30 天和 90 天统计窗口，展示智能体运行、工作流运行、Token 消耗、模型调用和成果文件数量。
   “校友权益”概览展示活跃权益、过期权益、7 天内到期、总发放、剩余额度和预留额度，用于判断 100 万 Token 活动的消耗速度和补发压力。
2. 热门智能体和热门工作流来自服务端审计日志，可用于决定优先优化或下架整改。
3. 最近运行记录只显示运营必要字段，不展示用户输入正文，避免在后台泄露教学内容。
4. 审计日志面板展示最近平台操作、操作者、对象和脱敏元数据，用于追溯智能体、工作流、账号、校友审核和备份恢复变更。

## 用户反馈处理

1. 登录用户在“意见反馈”中提交功能建议、使用问题、智能体接入或其他反馈。
2. 反馈写入服务端队列和审计日志，不再只停留在浏览器。
3. 管理员在后台查看反馈，标记为处理中或已关闭，并填写处理备注。
4. 反馈数据进入平台备份，便于上线迁移、复盘和服务质量评估。

## 账号管理

1. 首期本地账号由超级管理员在管理后台创建和停用。
2. 管理员只设置初始密码或重置密码，不在前端保存密码；初始和重置密码至少 8 位，用户首次登录后仍需自行修改。
3. 至少保留 1 个启用状态的超级管理员；服务端会拒绝停用或降权最后一个 `super_admin`。
4. 正式接入学校统一身份认证后，本地角色仍作为平台授权层保留。
5. 超级管理员可在“账号与角色”面板导出账号台账 CSV，字段包含账号、角色、部门、状态、SSO 来源和首次改密状态；导出不包含密码哈希、会话或密钥，并写入 `user.export` 审计日志。

## 备份与恢复

1. 超级管理员可在管理后台导出平台备份，用于迁移或误操作回滚。
2. 备份只包含配置和运营元数据，不包含密码哈希、会话 token 或模型密钥。
3. 恢复时服务端会拒绝未来版本备份和含 `passwordHash`、`sessions`、`tokenReservations`、`apiToken`、`secret`、`key` 等敏感字段的备份。
4. SQLite 数据库、上传证明和成果 HTML 文件所在的 `DATA_DIR` 目录仍需要定期做文件级加密备份；Linux systemd 部署可启用 `deploy/systemd/xiaozhuang-platform-backup.timer`，每天把 `/var/lib/xiaozhuang-platform` 归档到 `/var/backups/xiaozhuang-platform` 并生成 SHA-256 校验文件。
5. 每次留档应同时保存后台平台备份 JSON、JSON manifest 校验值、DATA_DIR 归档和 `.sha256` 文件；恢复时先恢复文件级数据目录，再通过后台恢复平台 JSON 备份。
6. 恢复操作会覆盖当前智能体、工作流和运营数据，执行前应先导出现状备份。

## 智能体测试状态筛选

100 个智能体批量治理时，管理员可在“应用管理”中使用“测试状态”筛选，快速定位已通过、未测试、测试失败和发布受阻的应用。外部网页、嵌入页和 API 型智能体若处于草稿或下架状态且未通过连接测试，会进入“发布受阻”范围，应先完成单个或批量测试，再执行发布。

## 智能体目录质量治理

“应用管理”的治理概览会按当前筛选范围显示目录质量，统计已达标和待补应用。管理员可用“目录质量”下拉直接筛选“目录达标”或“待补资料”，每行应用会提示主要缺口，例如缺 Logo、简介过短、缺负责人、系统提示词过短、推荐问题少、模型路由无效、外部应用地址缺失或 API 地址缺失。管理员也可搜索这些缺口关键词，再批量分派给应用负责人补齐。

导出智能体目录时，JSON 会附带 `qualitySummary`、`qualityStatus`、`qualityIssueCount`、`qualityIssues` 和 `qualityLabel`，适合作为整改台账发给学院或应用负责人。再次导入时，平台会剥离这些派生字段，只保留正式智能体配置。

超级管理员还可以点击“质量台账”导出服务端 CSV，字段包含智能体 ID、名称、分类、负责人、发布状态、接入方式、模型路由、最近测试状态、质量状态、缺口数量和具体缺口。该导出会写入 `agent.quality.export` 审计日志。

## 工作流测试状态筛选

教学工作流治理时，管理员可在“工作流批量操作”区域按测试状态筛选流程。筛选后，“全选工作流”、批量测试和批量模型调整只针对当前可见范围，便于先处理未测试、测试失败或发布受阻的流程，再进入正式发布。

## 反馈导出留档

超级管理员可在“反馈处理”面板导出反馈 CSV，字段包含提交时间、用户、类型、内容、邮箱、处理状态、处理人和备注。导出会写入 `feedback.export` 审计日志，适合用于阶段复盘、需求整理和上线后问题闭环。

## 校友认证导出留档

超级管理员可在“校友认证审核”面板导出校友认证 CSV，字段包含申请人、毕业年份、院系专业、联系方式、证明文件 ID、人工/联合认证状态、审核人、通过或退回时间、权益发放金额和到期时间。导出会写入 `alumni.export` 审计日志，用于和校友会接口切换、人工审核复核及 Token 权益结算留档。

## 校友额度人工调整

超级管理员可在“额度人工调整”面板对已有活跃校友权益进行补发或扣减，用于误扣补偿、活动额度或人工纠偏。扣减不能超过当前剩余额度；所有调整都会写入 Token 账本和 `token.adjust` 审计日志。

## 审计日志筛选

管理员可在“审计日志”面板按关键词和动作类型筛选最近审计记录，用于快速定位智能体变更、工作流发布、校友审核、Token 调整、成果删除、备份恢复和登录异常。超级管理员仍可导出完整脱敏审计 CSV 留档。
筛选后，审计治理概览会同步显示当前范围内主要动作、操作者数量、系统事件和失败/删除/恢复相关风险线索，便于快速判断是否需要进一步追溯。

## 模型调用台账筛选

管理员可在“模型调用台账”面板按关键词、调用状态和模型路由筛选最近模型运行记录，用于排查供应商故障、备用链路触发、Token 结算异常和特定智能体或工作流的模型表现。超级管理员仍可导出脱敏模型调用 CSV 留档。
筛选后，台账治理概览会同步显示当前范围内成功/失败、备用链路触发次数、Token 消耗和平均延迟，适合快速判断某个模型、工作流或时间段是否需要降级、切备用或暂停发布。

## 反馈队列筛选

管理员可在“反馈处理”面板按关键词、处理状态和反馈类型筛选用户反馈，用于定位高频问题、跟踪处理中事项和整理上线后需求。超级管理员仍可导出反馈 CSV 留档。
筛选后，反馈治理概览会同步显示当前范围内待处理、处理中、已关闭、主要反馈类型和是否留有邮箱，便于安排回访与上线后问题分派。

## 校友认证审核筛选

审核员可在“校友认证审核”面板按关键词和认证状态筛选申请，默认优先显示待审核记录，也可切换查看全部、已通过和已退回记录，用于复核联合认证结果、人工审核过程和 Token 权益发放留痕。
筛选后，审核治理概览会同步显示当前范围内待审、通过、退回、联合认证状态和待审最高权益预估，便于安排审核人手并预判 100 万 Token 权益发放规模。
## 上线自检筛选
管理员可在“上线自检”面板按关键词和状态筛选检查项，优先处理生产密码、模型 Key、Coze Workflow、校友会接口、统一身份认证、官方素材和部署产物等阻断项。上线前建议先切换到“只看阻断”，逐项消除后再查看“提醒”项并更新交接说明。

上线自检也会汇总“智能体质量治理”和“工作流质量治理”缺口。若出现提醒项，管理员应先导出对应质量台账，再按负责人分派补齐 Logo、简介、提示词、推荐问题、路由、输入输出规范、质检清单和测试状态。
需要向不同责任人分派上线事项时，可使用“责任方”下拉、关键词或状态筛选，再点击“导出当前”生成 CSV 清单；导出内容包含检查项名称、分组、责任方、等级和说明，不包含任何密钥。部署自检报告也会带出责任方，便于分派给云平台运维、模型服务、校友会接口、统一身份认证、品牌素材和应用/工作流管理员。
## 模型路由状态筛选
管理员可在“模型路由状态”面板按服务商、能力、模型名称、配置状态和模型类型筛选供应商，并通过治理概览查看当前范围内已配置/未配置、国产/国际和可批量测试数量。上线前建议先切换到“未配置”，补齐 DeepSeek、Kimi、ChatGPT、Coze、Gemini 等关键 Key，再分别查看国产模型和国际模型的连接测试结果。
补齐 Key 后可点击“测试已配置”批量测试服务端已配置的模型供应商。批量测试只针对服务端密钥状态，不会把 API Key 写入前端或导出文件；测试不走备用链路、不扣用户额度，会逐个写入模型调用台账和审计日志。若部分供应商失败，应先处理失败项再进入正式开放。
## 账号角色筛选
超级管理员可在“账号与角色”面板按关键词、角色和状态筛选账号，用于快速定位本地账号、统一认证账号、应用管理员、校友审核员和停用账号。正式接入学校 SSO 后，建议定期筛选“校友审核员”和“应用管理员”角色，复核权限是否仍与岗位一致。
筛选后，账号治理概览会同步显示当前范围内启用/停用、角色分布、统一认证/本地账号和待改密数量，便于上线初期排查默认密码、冗余管理员和 SSO 迁移进度。
## 智能体目录排序
管理员可在“应用管理”中按默认顺序、名称、分类、状态、模型路由和测试状态排序 100 个智能体。批量发布、批量下架、批量设模型和批量测试仍只作用于当前筛选与排序后的可见范围，便于分批治理同一学院、同一模型或同一测试状态的智能体。
筛选或排序后，“当前范围”治理概览会同步显示可见智能体数量、发布状态、测试状态和主要模型路由分布，用于判断这一批智能体是否已经具备发布条件。
需要把部分智能体交给学院或项目组复核时，可使用“导出当前”只导出当前筛选和排序后的可见智能体；原“导出”按钮仍会优先导出已勾选智能体，其次导出筛选范围或全部目录。
## 工作流检索与排序
管理员可在“教学工作流管理”中按关键词检索流程标题、摘要、首选模型、成果类型和测试状态，也可用“流程质量”下拉筛选流程达标或待补流程，并按名称、模型路由、成果类型、发布状态或测试状态排序。全选、批量测试和批量模型调整只作用于当前检索与排序后的可见流程，适合分批维护备课、教案、课件、题目、资源和网页动画工作流。
工作流治理概览会随当前检索、质量筛选和排序结果同步更新，显示可见流程数量、发布状态、测试状态、主要模型路由、成果类型分布和流程质量缺口；每行工作流也会显示“流程达标”或主要待补项，用于判断这一批流程是否适合进入批量测试、模型调整或导出。
需要给教研组或外部审核人员交接部分流程时，可使用“导出当前”只导出当前检索与排序后的可见工作流；导出的 JSON 仍保持与“导入”兼容，可选择合并或替换。
超级管理员可点击“流程台账”导出工作流质量 CSV，字段包含工作流 ID、名称、发布状态、首选模型、成果类型、最近测试状态、质量状态、缺口数量、具体缺口、输入字段数、输出结构数、质检清单数和提示词长度；导出会写入 `workflow.quality.export` 审计日志。
